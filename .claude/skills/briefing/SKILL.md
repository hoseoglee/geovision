# Briefing Skill

웹에서 뉴스/정보를 검색하여 TTS 친화적으로 요약한 뒤, 음성 또는 텍스트로 브리핑한다.

## 사용법

```
/briefing [주제]              # 주제별 뉴스 브리핑 (기본: 주요 뉴스)
/briefing 경제 뉴스            # 경제 뉴스 브리핑
/briefing --text              # 음성 없이 텍스트만 출력
```

## 검증 기준

- [ ] 3~5개 핵심 뉴스를 선별하여 요약
- [ ] 각 뉴스당 2~3문장 (100~150자) 이내
- [ ] TTS로 읽었을 때 자연스러운 한국어 문체
- [ ] /api/speak로 전송 성공 (--text 아닌 경우)
- [ ] 전체 브리핑 2~3분 분량 (400~600자)

## 워크플로우

### 1. 주제 파악
- 사용자 입력에서 주제 키워드 추출
- 주제 미지정 시 "오늘의 주요 뉴스"로 기본 설정
- `--text` 플래그 감지 → TTS 전송 생략

### 2. 뉴스 수집 (병렬 WebSearch)
- 병렬로 2~3개 검색 쿼리 실행:
  ```
  ├─ WebSearch: "{주제} 뉴스 오늘 {날짜}"
  ├─ WebSearch: "{주제} 최신 동향 {날짜}"
  └─ WebSearch: "{관련 키워드} 속보" (필요 시)
  ```
- 각 검색에서 상위 3~5개 결과 수집
- WebFetch로 주요 기사 본문 확인 (필요 시)

### 3. 뉴스 선별 및 순위
- **선별 기준**: 영향력 > 시의성 > 독창성
- 중복 기사 제거
- **최종 3~5개** 뉴스 선정

### 4. TTS 친화적 스크립트 작성
- **구조**:
  ```
  [인트로] 안녕하세요, {날짜} {주제} 브리핑입니다.

  [뉴스 1] 첫 번째 소식입니다. {핵심 내용 2~3문장}
  [뉴스 2] 다음 소식입니다. {핵심 내용 2~3문장}
  [뉴스 3] 세 번째 소식입니다. {핵심 내용 2~3문장}

  [아웃트로] 이상 {주제} 브리핑이었습니다.
  ```

- **TTS 최적화 규칙**:
  - 문장당 15~20단어 이내 (자연스러운 호흡 단위)
  - 능동태 사용: "삼성이 발표했습니다" (O) vs "발표가 되었습니다" (X)
  - 약어 풀어쓰기: "GDP" → "지디피", "API" → "에이피아이"
  - 숫자는 문맥에 맞게: "3.5%" → "3.5퍼센트", "2026년" → 그대로
  - 영문 고유명사는 한글 병기: "테슬라", "애플"
  - 괄호, 대시, 복잡한 구문 회피
  - 뉴스 간 전환어 사용: "다음 소식입니다", "한편", "마지막으로"

### 5. 전달
- **음성 모드** (기본):
  1. 스크립트 텍스트를 사용자에게 먼저 출력
  2. `/api/speak`로 POST 전송:
     ```bash
     curl -s -X POST http://localhost:3001/api/speak \
       -H 'Content-Type: application/json' \
       -d '{"text":"브리핑 스크립트 전문"}'
     ```
  3. 전송 성공 확인
- **텍스트 모드** (`--text`):
  - 스크립트만 출력, TTS 전송 생략

## 제약 조건

- 깊이 있는 분석/비교는 하지 않음 (→ research 스킬)
- 브리핑 저장/아카이빙은 하지 않음 (일회성 소비 콘텐츠)
- 실시간 속보 추적은 하지 않음 (단발성 요약)
- 뉴스 원문 전체를 읽어주지 않음 (요약만)

## 핵심 원칙

- **간결함 우선**: 읽는 사람이 아닌 듣는 사람을 위한 글쓰기
- **역피라미드**: 가장 중요한 정보를 먼저
- **자연스러운 말투**: 뉴스 앵커가 읽는 것처럼 자연스럽게
- **속도 > 깊이**: 빠른 파악이 목적, 심층 분석은 research로

## 스킬 의존성

- **사용하는 도구**: WebSearch, WebFetch, Bash (curl)
- **연관 스킬**: research (심층 조사 시 안내), work_report (작업 보고)
