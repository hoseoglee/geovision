# Research Skill

주제를 조사하고 구조화된 리서치 보고서를 생성한다. `/research <주제>` 또는 자연어로 호출.

## 워크플로우

### 1. 주제 파악
- 사용자 요청에서 조사 주제와 범위를 파악한다.
- 불명확하면 간결하게 확인 질문을 한다.
- 조사 깊이를 판단한다:
  - **quick**: 단일 주제, 핵심 정보만 (검색 2-3회)
  - **standard**: 여러 관점, 비교 분석 포함 (검색 5-8회) — 기본값
  - **deep**: 종합 분석, 다수 소스 교차 검증 (검색 10회+)
- 사용자가 `/research deep <주제>`, `/research quick <주제>`로 깊이를 지정할 수 있다.

### 2. 자료 수집
- **병렬 처리**: 독립적인 검색은 Agent 도구로 병렬 수행한다.
- 수집 소스:
  - **웹 검색** (WebSearch): 최신 정보, 공식 문서, 블로그, 논문
  - **웹 페이지 읽기** (WebFetch): 검색 결과 중 유용한 페이지 상세 분석
  - **코드베이스 탐색**: 관련 코드/설정이 있으면 Glob, Grep, Read로 탐색
  - **기존 papyrus 문서**: ~/papyrus/ 내 관련 기존 자료 확인
- 각 소스에서 핵심 정보, 인용구, 데이터를 추출한다.
- **출처를 반드시 기록**한다 (URL, 문서 경로, 날짜).

### 3. 분석 및 종합
- 수집한 자료를 주제별로 분류한다.
- 공통점, 차이점, 상충되는 정보를 식별한다.
- 핵심 인사이트를 도출한다.
- 필요 시 비교 테이블, 장단점 분석을 포함한다.

### 4. 보고서 작성
- 아래 템플릿에 따라 보고서를 작성한다.
- 파일명 형식: `yyyy-mm-dd-hh-mm_{주제_한글_snake_case}.md` (예: `2026-03-07-22-30_토큰_효율화_조사.md`)
- 저장 위치: `~/papyrus/resources/`
- `~/papyrus/resources/` 디렉토리가 없으면 생성한다.

### 5. 동기화 및 보고
- 저장 후 `cd ~/papyrus && git add -A && git commit -m "research: {주제}" && git push origin main` 실행.
- 사용자에게 핵심 발견사항 요약과 파일 경로를 알려준다.

### 6. 작업 보고서
- 리서치 완료 후 반드시 `work_report` 스킬을 호출하여 작업 보고서를 생성한다.
- 리서치 보고서(resources/)와 작업 보고서(records/)는 별개: 리서치 보고서는 조사 결과물, 작업 보고서는 수행한 작업의 기록이다.

## 핵심 원칙

- **출처 명시**: 모든 주장/데이터에 출처를 달아 검증 가능하게 한다.
- **사실과 의견 구분**: 객관적 사실과 AI의 분석/의견을 명확히 구분한다.
- **병렬 수집**: 독립적인 검색은 Agent로 병렬 실행하여 속도를 높인다.
- **기존 자료 활용**: papyrus에 관련 문서가 있으면 중복 조사하지 않고 참조/확장한다.
- **실용성 우선**: 학술 논문이 아니라 실무에 도움되는 정보 중심으로 정리한다.
