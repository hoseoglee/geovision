# Pilot — AI 업무 대행 파일럿

> 회사 업무를 화면을 보면서 대신 처리하고, 쓸수록 똑똑해지는 시스템.

---

## 실행 흐름

```
사용자: "파일럿, 메일 보내줘"
         │
    [1] 컨텍스트 로드
         ├─ ~/papyrus/areas/업무PC_환경맵.md (PC_B 상태)
         ├─ ~/workspace/project_pilot/playbooks/ (학습된 절차)
         ├─ ~/.anima/soul/memory/lessons.md (관련 교훈)
         └─ ~/.anima/soul/memory/preferences.md (사용자 선호)
         │
    [2] 플레이북 탐색
         ├─ 있음 → 플레이북 기반 실행 (빠르고 정확)
         └─ 없음 → 자율 탐색 실행 (시행착오 허용)
         │
    [3] screen_agent로 실행
         ├─ KVM 접속 (IP 자동 감지)
         ├─ 스냅샷 → Vision 분석 → 액션
         └─ 매 액션마다 트레이스 기록
         │
    [4] 학습
         ├─ 성공 → 플레이북 생성/갱신
         ├─ 실패 → 교훈 기록
         └─ 환경 변화 → 환경맵 갱신
```

## 단계별 상세

### [1] 컨텍스트 로드

```bash
# CLI로 확인 가능
cd ~/workspace/project_pilot
python main.py context 메일    # "메일" 관련 컨텍스트 로드
```

환경맵에서 PC_B 상태(설치된 앱, IP, 워크스페이스)를 파악하고,
플레이북에서 해당 업무의 학습된 절차를 검색하고,
교훈에서 관련 실패 패턴을 확인한다.

### [2] 플레이북 탐색

```bash
python main.py playbooks           # 전체 목록
python main.py playbook 메일_보내기  # 상세 절차
```

- **확정(confirmed)** 플레이북: 2회+ 성공한 검증된 절차 → 그대로 따라 실행
- **후보(candidate)** 플레이북: 1회 성공한 절차 → 참고하되 유연하게 실행
- **없음**: screen_agent의 자율 탐색으로 수행

### [3] 실행 — 키보드 우선 (HID Only)

**IMPORTANT — PC_B와 네트워크 직접 통신 불가. HID(키보드/마우스)가 유일한 인터페이스.**
**마우스 좌표 클릭은 최후 수단. 키보드만으로 수행한다.**

```
실행 우선순위:
  1. AutoHotkey 단축키 (Ctrl+Alt+A → AWS, Ctrl+Alt+G → GitHub 등)
  2. Vimium (브라우저에서 f키 → 힌트 알파벳 입력)
  3. Windows 키보드 (Win+숫자, Ctrl+L, Tab, Alt, Enter)
  4. KVM 마우스 (큰 타겟만, 위 3가지 불가능할 때만)
```

**자율 시행착오 학습은 여전히 동작한다:**

**매 액션마다 기록하는 것:**
- 액션 전/후 화면 상태 변화 (뭐가 바뀌었는지)
- 실패 시: 뭘 시도했고, 왜 안 됐고, 대신 뭘 했고, 됐는지
- 성공 시: 어떤 방법이 효과적이었는지
- 새로 발견한 환경 정보 (앱 위치, UI 구조, 로딩 시간 등)

**폴백 시행착오 패턴:**
- 단축키 안 먹힘 → 마우스 클릭 시도 → 성공 시 "이 앱에서는 마우스가 나음" 학습
- 좌표 빗나감 → ±10px 보정 → 성공 시 정확한 좌표 학습
- 앱 안 열림 → 로딩 대기 추가 → 성공 시 "이 앱은 N초 대기 필요" 학습

### [4] 학습

**사용자 개입 없이 자동으로** 학습 프로세스를 수행한다:

1. **성공 시**:
   - 플레이북 없었으면 → 후보(candidate) 플레이북 자동 생성
   - 플레이북 있었으면 → 성공 횟수 +1, 2회 도달 시 확정(confirmed) 승격
   - 시행착오가 있었으면 → 최종 성공 경로만 플레이북에 저장 (실패 경로 제거)
2. **실패 시**:
   - 교훈을 lessons.md에 기록 (분류: ui-pattern / app-behavior / workflow)
   - 시행착오 과정도 기록 → 다음에 같은 실패 회피
3. **시행착오에서 배움**:
   - "A를 했는데 안 됐고, B를 했더니 됐다" → 자동 교훈: "A 대신 B를 먼저 시도할 것"
   - 폴백이 성공했으면 → effective 패턴으로 기록
   - 플레이북 있었으면 → 실패 횟수 +1
3. **환경 변화 발견 시**:
   - 업무PC_환경맵.md 자동 갱신
