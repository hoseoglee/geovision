# Brainstorm Skill

새로운 아이디어를 발굴하는 것이 핵심 목표. 외부 세계를 적극 탐색하고, 다양한 관점에서 발산한 뒤, 기존에 없는 조합을 찾아낸다.

## 사용법

```
/brainstorm <주제> [--method <기법>] [--count <목표 수>] [--depth shallow|deep]
```

- `<주제>`: 브레인스토밍할 주제 또는 질문
- `--method`: 특정 기법 지정 (기본: auto — 주제에 맞게 자동 선택)
- `--count`: 목표 아이디어 수 (기본: 30)
- `--depth shallow|deep`: shallow=탐색+발산만, deep=전체 파이프라인 (기본: deep)

## 워크플로우

```
Phase 1: FRAME ─── 문제를 날카롭게 재정의
    │
Phase 2: EXPLORE ─ 외부 탐색 (웹 서치, 트렌드, 타 분야 사례) ★ 핵심
    │
Phase 3: DIVERGE ─ 병렬 에이전트로 대량 발산
    │
Phase 4: CROSS ─── 교차 수분 — 서로 다른 결과를 강제 조합
    │
Phase 5: EVALUATE ─ 신규성 중심 평가
    │
Phase 6: REFINE ── 실행 가능한 수준으로 구체화
```

### Phase 1: FRAME (문제 정의) — 5%

주제를 브레인스토밍에 적합한 질문으로 재구성한다.

1. 주제의 핵심 의도 파악
2. "How Might We(HMW)" 질문 **3개** 변환 (넓은 것, 좁은 것, 도발적인 것)
   - 넓은: "어떻게 하면 사용자 경험을 근본적으로 바꿀 수 있을까?"
   - 좁은: "어떻게 하면 첫 사용 후 3일 내 재방문율을 50% 올릴 수 있을까?"
   - 도발적: "앱을 없애도 같은 가치를 전달할 수 있다면?"
3. 범위와 제약 조건 명시
4. **금지어 설정**: 이미 뻔한 답을 명시적으로 배제 ("~은 이미 알고 있으니 제외")

**출력**: HMW 질문 3개 + 범위 + 금지어 목록

### Phase 2: EXPLORE (외부 탐색) — 25% ★

**새로운 아이디어의 원료를 외부에서 가져오는 핵심 단계.** AI 내부 지식만으로는 진정한 신규 아이디어가 나오기 어렵다.

#### 병렬 탐색 에이전트 (3~4개 동시)

```
Agent 탐색-A: 트렌드/신기술 스캔
  → WebSearch로 최신 트렌드, 신기술, 시장 동향 조사
  → "2025~2026 {주제 분야} trends", "{주제} emerging technology"
  → 아직 메인스트림이 아닌 초기 단계 기술/서비스에 주목

Agent 탐색-B: 타 분야 사례 수집 (Cross-pollination)
  → 주제와 전혀 다른 산업/분야에서 유사 문제를 어떻게 해결했는지 탐색
  → 예: 교육 문제 → 게임 산업, 의료, 군사 훈련, 스포츠 코칭의 해결법 참고
  → "how {다른 분야} solved {유사 문제}", "{주제} inspired by {자연/역사/예술}"

Agent 탐색-C: 사용자 불만/니즈 수집
  → 기존 솔루션의 불만, 개선 요청, 미충족 니즈 탐색
  → Reddit, 커뮤니티, 리뷰, GitHub Issues 등에서 실제 사용자 목소리
  → "{주제} complaints", "{주제} wishlist", "{기존 솔루션} alternatives"

Agent 탐색-D: 학술/특허 탐색 (deep 모드에서만)
  → 최근 논문, 특허에서 아직 상용화되지 않은 아이디어 발굴
  → arxiv, Google Scholar 등
```

#### 탐색 출력물
각 에이전트가 수집한 결과를 **영감 카드(Inspiration Card)** 형태로 정리:

```
[영감 카드 #1]
출처: {URL 또는 분야}
핵심: {한 줄 요약}
전이 가능성: {우리 주제에 어떻게 적용 가능한지}
```

**출력**: 영감 카드 10~20장

### Phase 3: DIVERGE (발산) — 30%

Phase 2의 영감 카드를 씨앗으로 삼아, **병렬 에이전트로 다양한 기법을 동시 실행**한다.

#### 기법 메뉴 (auto 모드 시 주제에 따라 3개 자동 선택)

| 기법 | 적합한 상황 | 설명 |
|------|------------|------|
| `scamper` | 기존 제품/서비스 개선 | Substitute, Combine, Adapt, Modify, Put to other use, Eliminate, Reverse |
| `6hats` | 복잡한 의사결정 | De Bono의 6색 모자 — 사실/감정/비판/낙관/창의/관리 관점 순회 |
| `reverse` | 고착된 문제 | 역발상 — "어떻게 하면 최악으로 만들 수 있을까?" → 반전 |
| `analogy` | 혁신적 아이디어 필요 | 타 산업/자연/역사에서 유사 패턴 차용 (Phase 2 영감 카드 활용) |
| `constraint` | 현실적 아이디어 필요 | 극단적 제약 — "예산 0원", "1시간 안에", "코드 10줄로", "오프라인만" |
| `persona` | 사용자 중심 문제 | 극단적 페르소나 — 5세 아이, 시각장애인, 미래인, 원시인, 외계인 |
| `random` | 막혀있을 때 | 무작위 단어/이미지 자극으로 강제 연결 |
| `triz` | 기술적 모순 해결 | 40가지 발명 원리 중 관련 원리 적용 |
| `future` | 전략/비전 수립 | "10년 후 이것이 사라졌다면 이유는?", "AI가 99% 자동화하면?" |
| `worst` | 아이디어 품질 높이기 | 최악의 아이디어 10개 → 왜 나쁜지 분석 → 정확히 반전 |
| `mashup` | 신규 조합 발굴 | 영감 카드 2장을 강제 결합 — "A와 B를 합치면?" |

#### 발산 규칙
- **양 우선**: 판단을 유보하고 최대한 많이 생산 (Serial Order Effect — 후반부가 더 독창적)
- **돌발 환영**: 비현실적/황당할수록 좋다 — 나중에 현실화하는 것이 처음부터 현실적인 것보다 새롭다
- **영감 카드 참조 필수**: 각 에이전트는 Phase 2 영감 카드를 반드시 참조하여 외부 자극을 반영
- **독립 실행**: 각 에이전트는 다른 에이전트의 결과를 모른 채 독립 생성 (LLM 편향 방지)
- **각 기법당 10개 이상** 생성, 앞쪽 5개는 워밍업으로 간주

#### 병렬 실행 구조
```
Agent 발산-A: 기법 1 + 영감 카드 홀수번 → 아이디어 10~15개
Agent 발산-B: 기법 2 + 영감 카드 짝수번 → 아이디어 10~15개
Agent 발산-C: 기법 3 + 영감 카드 전체 → 아이디어 10~15개
```

**출력**: 번호가 매겨진 전체 아이디어 목록 (기법별 그룹, 총 30~45개)

### Phase 4: CROSS (교차 수분) — 10%

**서로 다른 에이전트의 결과를 강제로 교차 조합**하여 제3의 아이디어를 만든다.

1. Agent A의 상위 아이디어 3개 × Agent B의 상위 아이디어 3개 → 강제 결합
2. Agent B의 상위 3개 × Agent C의 상위 3개 → 강제 결합
3. 가장 다른(거리가 먼) 두 아이디어를 골라 결합 → "이 둘을 동시에 만족시키려면?"
4. 영감 카드 중 사용되지 않은 것과 기존 아이디어 결합

**출력**: 교차 조합 아이디어 5~10개 추가

### Phase 5: EVALUATE (평가) — 15%

**신규성(Novelty)에 가중치를 두고** 평가한다.

#### 평가 매트릭스 (각 1~5점, 가중치 적용)

| 기준 | 가중치 | 설명 |
|------|--------|------|
| **Novelty** | ×3 | 기존에 없는 새로운 접근인가? (WebSearch로 유사 사례 존재 여부 확인) |
| **Impact** | ×2 | 문제 해결 효과, 가치 창출 규모 |
| **Feasibility** | ×1 | 기술적/자원적 실현 가능성 |
| **Speed** | ×1 | 구현/실행 속도 |

#### 신규성 검증 (Novelty Check)
상위 후보 10개에 대해 WebSearch로 유사 사례 확인:
- 이미 존재하는 제품/서비스 → Novelty 1점
- 유사하지만 차별점 있음 → Novelty 3점
- 검색해도 유사 사례 없음 → Novelty 5점

#### 분류
- **New & Now**: Novelty ≥ 4 AND Feasibility ≥ 3 → 새롭고 실현 가능 ★ 최우선
- **Moonshot**: Novelty ≥ 4 AND Impact ≥ 4 AND Feasibility ≤ 2 → 도전적이지만 가치 큼
- **Quick Win**: Impact ≥ 3 AND Feasibility ≥ 4 → 즉시 실행 가능 (새롭진 않아도)
- **Drop**: 총점 하위 30% → 제외

**출력**: 평가 테이블 + 분류별 그룹

### Phase 6: REFINE (정제) — 15%

상위 아이디어를 실행 가능한 수준으로 구체화한다.

1. **Top 5 아이디어** 선정 (New & Now 우선)
2. 각 아이디어별:
   - 한 줄 설명 + 왜 새로운지(기존 대비 차별점)
   - 구체적 실행 방법 (3~5 단계)
   - 필요 자원/도구
   - 예상 장애물과 대응
   - **프로토타입 방법**: 최소 비용으로 검증하는 방법
   - 첫 번째 액션 아이템 (지금 바로 할 수 있는 것)
3. **조합 제안**: Top 5 중 2개를 합치면 더 좋은 아이디어가 되는 조합 제안

**출력**: 정제된 아이디어 카드 + 조합 제안

## 출력 형식

### shallow 모드
Phase 1~3만 실행. 영감 카드 + 아이디어 목록 도출.

### deep 모드 (기본)
Phase 1~6 전체 실행. 최종 출력:

```markdown
## Brainstorm: {주제}

### HMW 질문
1. (넓은) ...
2. (좁은) ...
3. (도발적) ...

### 영감 카드 (외부 탐색)
{출처별 영감 카드 — 총 N장}

### 발산 결과
{기법별 아이디어 목록 — 총 N개}

### 교차 조합
{강제 결합으로 나온 제3의 아이디어}

### 평가
| 순위 | 아이디어 | Novelty(×3) | Impact(×2) | Feasibility | Speed | 총점 | 분류 |
|------|---------|-------------|------------|-------------|-------|------|------|

### Top 아이디어
{정제된 아이디어 카드 5개 — 차별점/프로토타입 방법 포함}

### 조합 제안
{Top 아이디어 간 시너지 조합}

### Next Action
{가장 먼저 해야 할 한 가지}

### Next Skill
{후속 스킬 제안: research / create_project / sdd spec / decompose}
```

## 결과물 저장

- **Phase 6 완료 후 반드시 보고서를 자동 저장**한다 (확인 없이 즉시 저장)
- 저장 경로: `~/papyrus/resources/{날짜}_brainstorm_{주제 요약}.md`
- Papyrus 문서 규칙 준수: YAML frontmatter(`tags: [brainstorm, ideation]`, `date`), `## 참고자료 (References)` 포함
- 보고서에는 Phase 1~6 전체 결과를 포함 (HMW 질문, 영감 카드 요약, 발산 전체 목록, 교차 조합, 평가 테이블, Top 아이디어 카드, 조합 제안, Next Action)
- shallow 모드에서도 Phase 1~3 결과를 저장
- 저장 후 `cd ~/papyrus && git add -A && git commit -m "brainstorm: {주제 요약}" && git push origin main`으로 동기화
- Top 아이디어가 프로젝트로 전환되면, 해당 brainstorm 문서를 `[[]]`로 참조 링크

## 새로운 아이디어를 찾기 위한 원칙

1. **밖에서 가져와라**: AI 내부 지식은 학습 데이터의 재조합일 뿐. WebSearch로 최신 트렌드, 타 분야 사례, 사용자 불만을 반드시 탐색한다
2. **먼 곳에서 빌려와라**: 가장 혁신적인 아이디어는 전혀 다른 분야의 해법을 가져올 때 나온다 (Cross-pollination). 교육 문제에 게임 메커니즘, 의료 문제에 물류 최적화 등
3. **강제 결합하라**: 관련 없어 보이는 두 아이디어를 억지로 합칠 때 예상치 못한 돌파구가 나온다 (아인슈타인의 "조합적 놀이")
4. **제약을 걸어라**: "예산 0원", "코드 10줄", "앱 없이" 같은 극단적 제약이 뻔한 답을 차단하고 창의적 우회로를 강제한다
5. **뻔한 답을 먼저 배제하라**: 금지어/금지 접근법을 명시하여 이미 알려진 해법의 반복을 방지
6. **후반부에 주목하라**: Serial Order Effect — 초기 아이디어는 뻔하고 후기 아이디어가 독창적. 충분히 많이 생성하고 앞쪽은 워밍업 취급
7. **존재 여부를 확인하라**: "새로운 아이디어"라고 생각한 것이 이미 있을 수 있다. 상위 후보는 반드시 WebSearch로 유사 사례를 확인 (Novelty Check)

## 검증 기준

- [ ] 외부 탐색(WebSearch)이 3회 이상 수행됨
- [ ] 영감 카드가 10장 이상 수집됨
- [ ] HMW 질문이 넓은/좁은/도발적 3가지로 제시됨
- [ ] 최소 30개 이상의 아이디어 발산
- [ ] 3개 이상의 기법이 병렬로 적용됨
- [ ] 교차 조합(CROSS) 단계가 수행됨
- [ ] 상위 후보에 대해 Novelty Check(유사 사례 검색) 수행됨
- [ ] Top 아이디어에 "왜 새로운지"(차별점)가 명시됨
- [ ] Top 아이디어에 프로토타입 방법이 포함됨
- [ ] New & Now 분류 아이디어가 하나 이상 도출됨
