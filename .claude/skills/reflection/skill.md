# Reflection Skill (회고 시스템)

"사고" 또는 "반성"이라고 하면 5단계 회고 파이프라인이 자동으로 돌아간다.

## 자동 트리거 (사용자가 "반성해"라고 안 해도 스스로 돌려야 하는 경우)

다음 상황을 감지하면 **즉시** 5단계 회고 파이프라인을 자동 실행한다:

1. **사용자 지적/질책 어투**: "~해야지", "왜 안했어", "왜 그랬어", "~하라고", "~했어야지", "어떡함", "맨날", "이런식으로 하면"
2. **사용자 교정**: "그게 아니라", "아니", "틀렸어", "아까도 말했는데"
3. **사용자 되물음**: "~했음?", "~한거야?", "~은?" — AI가 스스로 해야 할 일을 사용자가 확인해야 했다는 신호
4. **완료 판정 후 문제 발견**: 사용자가 결과물에서 오류를 지적

지적을 받으면:
- 즉시 해당 문제를 수정한다
- 수정과 동시에 5단계 회고 파이프라인을 **경량 모드**로 실행한다 (보고서 저장은 생략 가능하지만, 교훈 기록 + 스킬 개선은 반드시 수행)
- 같은 세션에서 3회 이상 지적받으면 **전체 모드**로 깊은 회고 실행

## 회고 파이프라인 (5단계)

```
원인 분석 → 모든 것 살펴보기 → 재발 방지 → 시스템 개선 → 보고서 작성
```

---

### 1단계: 원인 분석

#### 1-1. 작업/사고 수집
- 현재 세션에서 수행한 작업, 발생한 사고, 사용자 교정 내역을 수집한다.
- 대화 히스토리, 수정한 파일, 생성한 파일, 에러 로그 등을 정리한다.
- 작업이 없으면 "아직 반성할 작업이 없습니다"라고 안내하고 종료한다.

#### 1-2. 근본 원인 파악
- **무엇이 발생했는가** — 증상/결과를 명확히 기술
- **왜 발생했는가** — 직접 원인 → 근본 원인까지 추적
- **항상 5 Whys 수행** (경량/중량 무관):
  1. **왜?** [직접 원인]
  2. **왜?** [그 원인의 원인]
  3. **왜?** [더 깊은 원인]
  4. **왜?** [구조적/시스템적 원인]
  5. **왜?** [근본 원인]
  → **근본 원인**: [한 문장 요약]

#### 1-3. 5가지 관점 자체 평가
각 관점에 대해 1-5점 척도로 평가하고 근거를 제시한다.

| 관점 | 평가 기준 |
|------|----------|
| **정확성** | 사실 오류 없는가? 요구사항을 정확히 반영했는가? |
| **완전성** | 빠진 부분, 엣지 케이스, 에러 처리가 충분한가? |
| **효율성** | 불필요한 작업 없었는가? 더 나은 접근법이 있었는가? |
| **안전성** | 보안 취약점, 기존 코드 파괴, 롤백 가능성은? |
| **유지보수성** | 읽기 쉬운가? 향후 변경이 용이한가? |

---

### 2단계: 모든 것 살펴보기

**IMPORTANT**: 일부만 보고 "확인 완료"라고 하지 않는다. 6개 카테고리를 전부 확인한다.

작업/사고와 관련된 모든 곳에서 문제를 찾고 **즉시 수정**한다.

#### 전수 조사 범위 (6개 카테고리)

| # | 범위 | 확인 내용 |
|---|------|----------|
| 1 | `~/.anima/` | soul, vessels, adapters, scripts, projects 설정 |
| 2 | `~/.claude/` | skills, settings, memory 파일 |
| 3 | `~/papyrus/` | records, projects, resources, archives, areas |
| 4 | `~/workspace/` | 코드, docker-compose, .env, Caddyfile 등 |
| 5 | Docker 런타임 | 컨테이너, 네트워크, 볼륨, 이미지 |
| 6 | 외부 서비스 | DuckDNS, GitHub, Trello 등 |

#### 조사 방법
- 관련 키워드로 각 카테고리를 Grep/Agent 병렬 탐색
- 발견된 문제는 **즉시 수정** (수정 가능한 것)
- 수정 불가능한 것은 목록화하여 보고서에 포함
- 수정 전후를 기록

### 3단계: 재발 방지

#### 3-1. 교훈 추출 및 lessons.md 기록
`~/.anima/soul/memory/lessons.md`를 읽고 다음을 수행한다:

- **잘한 점** → `effective` 분류로 기록
- **실수/비효율** → 해당 분류(`process`, `understanding`, `technical`, `communication`, `approach`)로 기록
- **사용자 교정이 있었으면** → 해당 내용을 교훈으로 기록
- **기존 교훈을 적용했으면** → 해당 항목에 `적용 확인: yyyy-mm-dd` 추가

교훈 형식:
```markdown
## {분류} | {제목}

- **날짜**: yyyy-mm-dd
- **상황**: {어떤 상황에서 발생했는지}
- **교훈**: {무엇을 배웠는지, 다음에 어떻게 할지}
- **심각도**: high/medium/low
```

#### 3-2. Pre-Check Guard 자동 생성
새 교훈을 기록했으면, 해당 실수를 방지할 Pre-Check Guard 항목을 lessons.md 하단의 `# Pre-Check Guards` 섹션에 추가한다.

자문: "이 교훈에서 pre-check를 만들 수 있는가?"
- YES → 적절한 `pre-check | {카테고리}` 섹션에 체크리스트 항목 추가
- NO → 건너뛰기

#### 3-3. 재발방지 체크리스트 (중량 작업/사고 시 필수)
5 Whys에서 도출된 재발방지 조치를 체크리스트로 변환:
- [ ] [다음에 같은 유형 작업 시 확인할 항목 1]
- [ ] [확인할 항목 2]
이 체크리스트는 lessons.md의 해당 교훈에 첨부하고, `_knowledge.md`에도 추가한다.

---

### 4단계: 시스템 개선

#### 4-1. 지침 점검 (Instruction Review)
다음을 자문한다:
- "이번 작업에서 지침(persona.md)에 추가할 내용이 있는가?"
- "우회/폴백을 사용했다면 지침에 문서화되어 있는가?"
- "사용자가 직접 해결 방법을 알려줬다면, 해당 정보가 지침에 있는가?"

추가가 필요하면 `~/.anima/soul/persona.md`를 수정한다.

#### 4-2. 교훈 승격 확인
- 같은 유형의 실수가 **2회 반복**되면 → persona.md 규칙으로 즉시 승격
- 교훈의 `적용 확인`이 **2회 이상** 누적되면 → persona.md 규칙으로 승격
- 승격 시: lessons.md에 `- **승격됨**: yyyy-mm-dd → persona.md {섹션명}` 표기

#### 4-3. 스킬 개선 확인
- 이번 사고/반성에서 스킬 자체의 문제가 발견되었는가?
- YES → 해당 스킬의 SSOT(`~/.anima/soul/skills/`)를 수정하고 `sync.sh push`

#### 4-4. 런북/Argos 연동 (해당 시)
- 교훈이 특정 프로젝트의 운영 장애에 관한 것이면 → `_knowledge.md` 런북에 대응 절차 추가
- 교훈이 모니터링으로 사전 감지 가능한 패턴이면 → Argos probe/rule 제안

#### 5-2. 동기화
- `cd ~/papyrus && git add -A && git commit -m "reflection: {작업명}" && git push origin main`
- lessons.md 또는 persona.md를 수정했으면 `~/.anima/sync.sh push`도 실행
- 동기화 실패 시에도 보고서 저장은 성공으로 처리한다.

#### 5-3. 텔레그램 알림
- work_report와 동일한 방식으로 보고서를 텔레그램으로 전송한다.
- `sendDocument` API로 보고서 파일 첨부 + 축약 캡션

#### 5-4. 결과 안내
- 사용자에게 5단계 수행 결과를 요약하여 안내한다.

## 핵심 원칙

- **솔직함**: 자기 평가는 솔직하게, 높은 점수를 남발하지 않는다.
- **구체성**: "좋았다/나빴다"가 아니라 구체적 근거를 제시한다.
- **전수 조사**: 일부만 보고 "확인 완료"라고 하지 않는다. 6개 카테고리 전부 확인한다.
- **즉시 수정**: 발견한 문제는 보고서에 기록만 하지 않고 즉시 수정한다.
- **체계적 학습**: 교훈 → Guard → 지침 → 스킬로 이어지는 연쇄를 반드시 수행한다.

- [ ] **시스템 개선**: 지침 점검 + 승격 확인 + 스킬/런북/Argos 검토
