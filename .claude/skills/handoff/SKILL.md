# Handoff Skill (세션 핸드오프)

세션 종료 시 현재 작업 상태를 스냅샷으로 저장하여 다음 세션이 끊김 없이 이어받을 수 있게 한다.

## 세션 시작 시 (자동)

1. `~/papyrus/handoffs/` 디렉토리에서 가장 최근 핸드오프 파일을 확인한다.
2. 최근 핸드오프가 존재하면 사용자에게 안내한다:
   - "이전 핸드오프가 있습니다: {파일명}. 내용을 확인하시겠습니까?"
3. 사용자가 확인을 요청하면 핸드오프 내용을 읽고 컨텍스트를 복원한다.

## 핸드오프 생성 워크플로우

사용자가 핸드오프를 요청하면 다음 단계를 수행한다:

### 1. 정보 수집

- 현재 세션의 대화 히스토리와 코드 변경사항을 리뷰한다.
- `~/.anima/vessels/local/{adapter}.yaml`에서 vessel 정보를 읽고, `python3 -m hermes_client instance get`으로 인스턴스 번호를 가져온다.

### 2. 핸드오프 문서 생성

다음 구조의 마크다운 문서를 작성한다:

```markdown
---
tags: [handoff]
date: yyyy-mm-dd
vessel: [vessel ID]
project: [프로젝트명]
---
# 세션 핸드오프

**Vessel**: [vessel ID]
**일시**: yyyy-mm-dd hh:mm
**프로젝트**: [프로젝트명 또는 작업 컨텍스트]

## 수행한 작업 요약
- [이번 세션에서 수행한 작업을 간결하게 요약]

## 미완료 작업
- [ ] [진행 중이었던 작업 1]
- [ ] [진행 중이었던 작업 2]
- [x] [완료된 작업 (참고용)]

## 중요 컨텍스트
- **발견한 이슈**: [디버깅 중 발견한 문제, 예상치 못한 동작 등]
- **주의사항**: [다음 세션에서 주의해야 할 점]
- **결정 사항**: [이번 세션에서 내린 기술적/설계적 결정]

## 다음 세션 우선순위
1. [가장 먼저 해야 할 작업]
2. [그 다음 작업]
3. [후순위 작업]

## 관련 파일
- `/absolute/path/to/file1`
- `/absolute/path/to/file2`

## 참고자료 (References)
- [[관련_papyrus_문서]]
- 없음
```

### 3. 파일 저장

- 프로젝트명을 파악한다 (현재 작업 디렉토리, 대화 내용 등에서 추론).
- 파일명 형식: `yyyy-mm-dd-hh-mm_핸드오프_{프로젝트명}.md`
- 저장 경로: `~/papyrus/handoffs/`
- 디렉토리가 없으면 `mkdir -p ~/papyrus/handoffs/`로 생성한다.

### 4. 최근 핸드오프 링크 갱신

- `~/.anima/soul/memory/` 디렉토리가 없으면 생성한다.
- 저장한 핸드오프 파일을 `~/.anima/soul/memory/last_handoff.md`에 복사한다.
  - `cp ~/papyrus/handoffs/{파일명} ~/.anima/soul/memory/last_handoff.md`
- 이를 통해 다음 세션 시작 시 자동으로 최근 핸드오프를 로드할 수 있다.

### 5. 동기화

- Papyrus 동기화: `cd ~/papyrus && git add -A && git commit -m "handoff: {프로젝트명}" && git push origin main`
- Anima 동기화: `~/.anima/sync.sh push`

### 6. 확인

- 사용자에게 핸드오프 저장 완료를 알리고, 파일 경로를 제공한다.
- 다음 세션에서 자동으로 안내될 것임을 알린다.

## 검증 기준

- [ ] 다음 세션이 컨텍스트를 이어받을 수 있는 충분한 정보 포함
- [ ] 미완료 작업 목록이 명시됨
